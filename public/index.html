Esta API fue desarrollada como proyecto evaluativo para la materia Laboratorio de Computación IV, de la TUP. UTN FRBB

Se utilizó NodeJS con Express para su desarrollo, y se encuentra deployada en Render.

URL API Deployada en servidor

Meals

Random Meals

Random meals devuelve una lista de 10 comidas random con los siguientes atributos:

{{url}}/api-express/v1/meals/random
  • idMeal
  • strMeal: nombre del plato
  • strArea: region
  • strMealThumb: link hacia un imagen del plato
  • strYoutube: link a un video de Youtube de la receta
  • strInstructions: instrucciones de la receta

Nota: este endpoint tarda unos segundos en procesar la request.

Get meal by ID

Get meal by ID permite recuperar un plato en particular, pasando su Id como parametro en la URL de la request

{{url}}/api-express/v1/meals/id/{id}

Nota: no hay una lista taxativa conteniendo todos los IDs. En principio cualquier ID mayor a 52764 y menor a 53076, incluyendo los extremos, funcionaria.

Filter meals

Filter meals permite aplicar un filtro a la request con query params. Hay dos posibilidades:

  • "By first letter" o segun la primera letra del nombre. Retorna una lista de comidas que comienzan con la letra indicada

    {{url}}/api-express/v1/meals/filter?f={letra}

  • "By category" o por categoria, retorna una lista de comidas dentro de la categoria indicada. Algunos ejemplos:

    • Breakfast
    • Desserts
    • Miscellanous
    • Pasta
    • Seafood
    • Starter
    {{url}}/api-express/v1/meals/filter?c={categoria}

Nota: Por el momento sólo se puede usar un filtro por request, es decir, filtrar por letra o por categoria

Drinks

Random Drinks

Random drinks devuelve una lista de 10 tragos aleatorios con los siguientes atributos:

{{url}}/api-express/v1/drinks/randomDrinks
  • idDrink
  • strDrink: nombre del trago
  • strInstructions: instrucciones para preparar el trago
  • strDrinkThumb: link hacia un imagen del plato
  • ingredients: una lista de 5 ingredientes con sus medidas

Nota: este endpoint tarda unos segundos en procesar la request.

Get drink by ID

ID Drink permite recuperar un trago en particular, pasando su Id como parametro en la URL de la request

  • "By category" o por categoria, retorna una lista de tragos dentro de la categoria indicada. Algunos ejemplos: 17253, 12186, 17241, 11662
